# Задача 2: Публикация товара на маркетплейсе

## Постановка задачи

Требуется реализовать функциональность в Личном кабинете продавца, которая позволит опубликовать товар на маркетплейсе (витрине товаров). Данный документ содержит верхнеуровневую постановку в формате User Story, подробное описание требований через Use Cases и диаграмму процесса.

---

## 1. User Story

### US-001: Публикация товара на маркетплейсе

**Как** продавец (владелец магазина)  
**Я хочу** опубликовать мой товар на маркетплейсе  
**Чтобы** увеличить видимость товара и привлечь больше покупателей

#### Критерии приемки

- [ ] Продавец может заполнить основную информацию о товаре (название, описание, цена, категория)
- [ ] Продавец может загрузить фотографии товара (минимум 1, максимум 10)
- [ ] Продавец может выбрать категорию товара из предопределенного списка
- [ ] Продавец может установить цену и скидку (если необходимо)
- [ ] Продавец может указать количество товара на складе
- [ ] Продавец может просмотреть предпросмотр товара перед публикацией
- [ ] Продавец может опубликовать товар одним кликом
- [ ] После публикации товар становится видимым на маркетплейсе для покупателей
- [ ] Продавец получает подтверждение о успешной публикации

#### Бизнес-ценность

- Увеличение каталога товаров на маркетплейсе
- Повышение активности продавцов
- Улучшение пользовательского опыта для продавцов
- Расширение возможностей монетизации платформы

#### Приоритет

**Высокий** — критическая функция для функционирования маркетплейса

## 2. Use Cases (Варианты использования)

### UC-001: Основной сценарий публикации товара

#### Участники

- **Актор (основной):** Продавец
- **Актор (вторичный):** Система маркетплейса

#### Предусловие

- Продавец авторизован в Личном кабинете
- Продавец имеет активный магазин на маркетплейсе
- Продавец имеет права на публикацию товаров

#### Основной сценарий

1. Продавец переходит в раздел "Мои товары" в Личном кабинете
2. Продавец нажимает кнопку "Добавить новый товар"
3. Система открывает форму создания товара с полями:
   - Название товара (обязательное, макс. 100 символов)
   - Описание товара (обязательное, макс. 5000 символов)
   - Категория товара (обязательное, выпадающий список)
   - Цена (обязательное, число с двумя знаками после запятой)
   - Скидка (опционально, процент от 0 до 100)
   - Количество на складе (обязательное, целое число)
   - Фотографии товара (обязательное, минимум 1, максимум 10)
   - Теги/ключевые слова (опционально)
   - Характеристики товара (опционально, в зависимости от категории)

4. Продавец заполняет все обязательные поля
5. Продавец загружает фотографии товара
6. Продавец нажимает кнопку "Просмотреть" для предпросмотра
7. Система отображает товар так, как он будет выглядеть на маркетплейсе
8. Продавец проверяет информацию и нажимает "Опубликовать"
9. Система проверяет корректность данных
10. Система сохраняет товар в базе данных со статусом "Опубликован"
11. Система отправляет уведомление продавцу о успешной публикации
12. Товар становится видимым на маркетплейсе для покупателей
13. Система перенаправляет продавца на страницу товара

#### Постусловие

- Товар опубликован на маркетплейсе
- Товар видим для всех покупателей
- Продавец может редактировать или удалять товар

#### Альтернативные сценарии

**A1: Продавец заполнил некорректные данные**
- На шаге 9 система обнаруживает ошибку в данных
- Система отображает сообщение об ошибке с указанием проблемного поля
- Продавец исправляет данные и повторяет попытку публикации

**A2: Продавец отменяет публикацию**
- На любом этапе продавец может нажать "Отмена"
- Система закрывает форму без сохранения данных
- Продавец возвращается в раздел "Мои товары"

**A3: Продавец сохраняет товар как черновик**
- На шаге 8 продавец нажимает "Сохранить как черновик" вместо "Опубликовать"
- Система сохраняет товар со статусом "Черновик"
- Товар не видим на маркетплейсе
- Продавец может вернуться и завершить публикацию позже

**A4: Ошибка при загрузке фотографий**
- На шаге 5 при загрузке фотографии возникает ошибка
- Система отображает сообщение об ошибке (например, "Файл слишком большой")
- Продавец может повторить попытку или выбрать другой файл

**A5: Превышен лимит товаров**
- На шаге 9 система проверяет, не превышен ли лимит товаров для этого продавца
- Если лимит превышен, система отображает сообщение "Вы достигли максимального количества товаров"
- Продавец должен удалить или архивировать существующий товар перед публикацией нового

### UC-002: Редактирование опубликованного товара

#### Участники

- **Актор (основной):** Продавец
- **Актор (вторичный):** Система маркетплейса

#### Предусловие

- Продавец авторизован в Личном кабинете
- Товар опубликован на маркетплейсе
- Продавец является владельцем товара

#### Основной сценарий

1. Продавец переходит в раздел "Мои товары"
2. Продавец находит товар в списке и нажимает "Редактировать"
3. Система открывает форму редактирования с текущими данными товара
4. Продавец изменяет необходимые поля
5. Продавец нажимает "Сохранить изменения"
6. Система проверяет корректность данных
7. Система обновляет информацию о товаре в базе данных
8. Система отправляет уведомление продавцу об успешном обновлении
9. Изменения становятся видимы на маркетплейсе

#### Постусловие

- Информация о товаре обновлена на маркетплейсе
- История изменений сохранена в системе

#### Альтернативные сценарии

**A1: Продавец отменяет редактирование**
- На шаге 5 продавец нажимает "Отмена"
- Система закрывает форму без сохранения изменений

**A2: Продавец пытается изменить недопустимое поле**
- На шаге 6 система обнаруживает, что продавец пытается изменить поле, которое нельзя редактировать
- Система отображает сообщение об ошибке

---

### UC-003: Удаление товара с маркетплейса

#### Участники

- **Актор (основной):** Продавец
- **Актор (вторичный):** Система маркетплейса

#### Предусловие

- Продавец авторизован в Личном кабинете
- Товар опубликован на маркетплейсе
- Продавец является владельцем товара

#### Основной сценарий

1. Продавец переходит в раздел "Мои товары"
2. Продавец находит товар в списке и нажимает "Удалить"
3. Система отображает диалог подтверждения: "Вы уверены, что хотите удалить товар?"
4. Продавец нажимает "Да, удалить"
5. Система удаляет товар из маркетплейса (переводит в статус "Удален")
6. Система отправляет уведомление продавцу об удалении
7. Товар больше не видим на маркетплейсе для покупателей

#### Постусловие

- Товар удален с маркетплейса
- История товара сохранена в системе (для аудита)

#### Альтернативные сценарии

**A1: Продавец отменяет удаление**
- На шаге 4 продавец нажимает "Отмена"
- Товар остается опубликованным

---

### UC-004: Просмотр статистики товара

#### Участники

- **Актор (основной):** Продавец
- **Актор (вторичный):** Система маркетплейса

#### Предусловие

- Продавец авторизован в Личном кабинете
- Товар опубликован на маркетплейсе

#### Основной сценарий

1. Продавец переходит в раздел "Мои товары"
2. Продавец нажимает на товар для просмотра деталей
3. Система отображает информацию о товаре и статистику:
   - Количество просмотров
   - Количество добавлений в избранное
   - Количество заказов
   - Средняя оценка товара
   - Отзывы покупателей

4. Продавец может просмотреть более подробную статистику, нажав "Подробная статистика"

#### Постусловие

- Продавец видит актуальную информацию о производительности товара

## 4. Ограничения и предположения

### Ограничения

- Максимальный размер изображения: 5 МБ
- Максимальное количество изображений: 10
- Максимальная длина названия товара: 100 символов
- Максимальная длина описания: 5000 символов
- Максимальное количество товаров на одного продавца: 1000 (может быть увеличено по запросу)

### Предположения

- Продавец имеет активный магазин на маркетплейсе
- Продавец авторизован в системе
- Категории товаров предопределены администратором
- Система поддерживает основные форматы изображений (JPG, PNG, WebP)

---

## 5. Критерии успеха

- Продавец может опубликовать товар за < 3 минут
- Товар становится видимым на маркетплейсе в течение < 1 минуты после публикации
- Минимум 95% успешных публикаций (без ошибок)
- Удовлетворенность пользователей ≥ 4.5/5 звезд
- Время отклика системы при публикации < 2 сек
